# ADR-0001: プログラミング言語の選定

## ステータス

承認済み (2026-02-19)

## コンテキスト

Rocket (Git GUIクライアント) の開発言語を選定する必要がある。
当初はGoを候補としていたが、以下の課題が明らかになった。

- GoのGUIフレームワーク (Wails / Fyne / Gio) はエコシステムの成熟度が低い
- Webフロントエンド技術 (HTML/CSS/JS) を活用してUIを構築したい

## 決定

**Rust** を採用する。

## 理由

- パフォーマンスとメモリ安全性を両立できる
- Tauri (Rust製GUIフレームワーク) により、Webフロントエンド技術でUIを構築できる
- Git操作ライブラリ (git2-rs, gitoxide) のエコシステムが充実している
- クロスプラットフォーム (macOS / Linux / Windows) 対応が容易
- シングルバイナリとして配布でき、バイナリサイズもElectron比で大幅に小さい

## 却下した選択肢

### Go

- GUIフレームワークの成熟度が低い (Wails / Fyne / Gio いずれも発展途上)
- Webフロントエンド技術との統合が限定的

### TypeScript (Electron)

- メモリ消費量が大きい
- バイナリサイズが肥大化する
- Git操作部分にネイティブパフォーマンスが必要な場面で不利

## 影響

- 既存ドキュメント (roadmap.md, features.md) のGo前提の記述をすべて更新する
- ディレクトリ構成やCI/CDもRust/Tauriベースに変更する
